# UAV/UGV 공방 공격 시나리오 (시퀀스 다이어그램 기반)

> 대상 다이어그램: `UxS(UAV/UGV) — 데이터링크/전술망 — GCS/UCS — C5I/COP` 간
> 임무명령·텔레메트리·센서·Health 데이터 흐름.
> 각 화살표(데이터 흐름)를 하나의 공격 표면으로 보고, 저장소에 구현된 red/blue 코드로
> 재현·검증한 결과를 함께 기록한다.

## 0. 다이어그램 → 공격 표면 매핑

| # | 다이어그램의 데이터 흐름 | 방향 | 공격 표면 | 구현 모듈 | 탐지기 |
|---|--------------------------|------|-----------|-----------|--------|
| A | 임무명령 / 경로 / 제어 요청 | GCS → UxS | **C2 명령 주입** | `attacks/c2_injection.py` | `CommandAnomalyMonitor` |
| B | 텔레메트리(위치·속도·배터리·상태) | UxS → GCS | **GNSS 스푸핑**(위치 위조) | `attacks/gnss_spoof.py` | `GnssInsCrossCheck` |
| C | 센서 데이터(영상·EO/IR·라이다·탐지) | UxS → GCS | **AI 적대적 예제**(인지 회피) | `attacks/perception/adversarial_patch.py` | `SensorConsensusMonitor` |
| D | Health 데이터(고장·온도·통신품질) | UxS → GCS | 상태 위조(공격 은닉) | (설계) | — |
| E | 데이터링크/전술망(전송 계층) | 양방향 | **재밍 / DoS** | `attacks/jamming_dos.py` | `LinkHealthMonitor` |
| F | 상위 임무 지시 / 상황정보 공유 | C5I ↔ GCS | 상위체계 주입·오염 | (설계) | — |
| G | 데이터링크/전술망(전송 계층) | 양방향 | **MITM 인터셉션**(도청→변조) | `attacks/mitm_intercept.py` | `InterceptionMonitor` |

핵심 원리: 공격은 **개별 데이터 흐름의 무결성·가용성·신뢰성**을 노리지만,
그 결과는 반드시 GCS/UCS가 관측하는 **공통 텔레메트리 서명**으로 드러난다.
방어 에이전트는 특정 오토파일럿에 종속되지 않고 이 공통 서명만으로 탐지한다.

## 시나리오 A — C2 명령 주입 (임무명령/제어 요청 위조)

**공격 서사.** 공격자가 데이터링크/전술망 상의 명령 업링크(GCS → UxS)에 접근한다.
MAVLink를 비롯한 다수 전술 데이터링크는 명령 무결성·발신자 인증이 약하다. 공격자는
정상 운용자가 보내지 않은 명령 — 강제 귀환(RTL), 모드 급변, 무장해제 — 을 주입해
임무를 교란하거나 기체 제어권을 탈취한다. 탐지를 흐리기 위해 짧은 시간에 다수의 모드
변경 명령을 **폭주(replay flood)** 시켜 정상 명령 사이에 위조 명령을 섞는다.

**관측 서명.** 주입 명령은 `HEARTBEAT.custom_mode` 급변과 `COMMAND_ACK` 빈도 증가로
텔레메트리에 나타난다. → 방어의 "예상치 못한 모드 전환" + "비정상 명령 빈도" 탐지.

**대응 플레이북.** 명령 서명검증/재인증 요구 → 위조 명령 차단 → 링크 전환.
모드 전환은 Medium(자동 조치), 명령 폭주는 High(운용자 승인).

---

## 시나리오 B — GNSS 스푸핑 (텔레메트리 위치 위조)

**공격 서사.** 실제 공격자는 위성 신호보다 강한 위조 RF를 방사해 수신기가 가짜 위치를
산출하게 만든다. 본 환경은 이를 프로토콜 수준(MAVLink `GPS_INPUT` 외부 GPS 주입)으로
정직하게 모델링한다. 공격자는 급변 탐지를 회피하려 오차를 **서서히 램프업**(0→150m)한다.
원시 GPS(`GPS_RAW_INT`)는 위조 좌표로 끌려가지만, EKF/INS 융합 위치
(`GLOBAL_POSITION_INT`)는 관성항법으로 잠시 참 경로를 유지한다.

**관측 서명.** 두 위치원의 **divergence 증가**가 스푸핑의 관측 가능한 서명이다.
→ 방어의 GNSS-INS 교차검증이 임계값(경고 30m / 위험 100m) 초과로 탐지·에스컬레이션.

**대응 플레이북.** GNSS 신뢰도 하향 → INS/비전 기반 항법 전환 → 안전속도 제한 →
(Critical) 즉시 귀환/정지 권고. 30~100m는 High(자동 아님, 인간 승인),
100m 이상은 Critical.

---

## 시나리오 C — AI 적대적 예제 (센서 데이터 인지 회피)

**공격 서사.** UxS가 GCS로 올리는 센서 데이터(영상·EO/IR·라이다·탐지 이벤트)의
탐지 파이프라인(YOLO 등)을 노린다. 공격자가 표적/장애물에 적대적 패치를 부착하거나
입력에 미세 섭동(PGD)을 가하면, 인지모델이 표적을 놓치거나 오인식한다.

**관측 서명.** AI 탐지 신뢰도 급락 + 다중 센서 간 표적 판단 불일치.
→ 방어의 센서/AI 합의 감시(`SensorConsensusMonitor.inject_ai_result`)가 탐지.
※ 이 신호는 텔레메트리만으로는 관측 불가하며 인지 계층에서 주입해야 한다.

**대응 플레이북.** 센서 재검증 → 다중 센서 교차검증 → AI 신뢰도 하향 → 인간 확인 요청.

**상태.** `torch`/`ultralytics` 의존성 필요(용량 큼)로 이번 라이브 테스트에서는 제외.
탐지기 로직은 구현·검증 가능하나 실제 이미지 파이프라인은 별도 설치 필요.

## 시나리오 E — 재밍 / DoS (데이터링크/전술망 저하)

**공격 서사.** RF 재밍은 데이터링크의 패킷 손실·지연·대역폭 저하를 유발한다. 본 도구는
텔레메트리 경로 중간에 삽입되는 UDP 릴레이로 동일 효과(패킷 드롭 80% + 지연 400±150ms)를
재현한다. 지속적 링크 저하는 제어 지연을 넘어 제어 상실로 이어질 수 있다.

**관측 서명.** HEARTBEAT 간격 급증 및 두절. → 방어의 링크 상태 감시가
간격 임계(정상×3) 초과는 Medium, 3초 이상 두절은 High로 탐지.

**대응 플레이북.** 자율 안전모드 유지 → 통신두절 안전정책(사전 정의 경로 복귀/정지) →
다중 링크 전환.

## 시나리오 G — 데이터링크 MITM 인터셉션 (도청 → 능동 변조)

**공격 서사.** 공격자가 데이터링크/전술망 상의 UxS↔GCS 경로에 중간자(MITM)로 삽입된다
(무선 릴레이 하이재킹, 악성 게이트웨이, 경로/ARP 오염 등). 본 환경은 이를 프로토콜
수준에서 차량 텔레메트리 포트와 GCS(방어) 사이에 삽입되는 UDP 릴레이로 정직하게
모델링한다. 공격은 두 단계다.

1단계 · **수동 도청.** 인터셉터는 모든 MAVLink 프레임을 바이트 단위로 그대로 중계하며
복호·파싱해 위치·비행모드·배터리·무장상태·명령이력을 탈취한다. 프레임을 변경하지
않으므로 텔레메트리 '내용'상 서명이 없다 — 도청은 본질적으로 은밀하며, 이 단계의 피해는
순수 기밀성 상실이다. (탐지는 링크 인증/암호화·네트워크 계층의 몫이다.)

2단계 · **능동 변조.** 공격자가 상황인식을 오염시키려는 순간 은밀성은 깨진다. 인터셉터가
GLOBAL_POSITION_INT(운용자가 신뢰하는 융합 위치)의 좌표에 오프셋을 주입하되 속도 필드
(vx,vy)는 그대로 둔다.

**관측 서명.** (1) 위치 변화량이 보고된 속도와 어긋남(운동학 정합성 위반), (2) 재기록된
프레임이 새 MAVLink 시퀀스로 방출되어 동일 sysid의 seq가 역행·급점프(발신원 정합성 위반),
(3) 위조된 융합위치 vs 참(true) 원시 GPS 간 divergence. → 방어의 `InterceptionMonitor`가
(1)(2)를, `GnssInsCrossCheck`가 (3)을 탐지(교차검증).

**대응 플레이북.** 링크 무결성(서명/시퀀스) 검증 → 텔레메트리 발신원 재인증 → 융합위치
신뢰도 하향·원시GPS/INS 교차검증 → 링크 암호화·경로 무결성 점검 → 링크 전환.
운동학 잔차 6~25m는 High, 25m 이상은 Critical.

### 테스트 중 발견한 부작용 (방어 로직 개선 후보)

재밍 테스트(E) 구간에서 **아무 위치 위조를 하지 않았는데도** GNSS-INS 교차검증이
39m·42m divergence를 High로 오탐(2건)했다. 원인: 80% 패킷 손실 하에서 `GPS_RAW_INT`와
`GLOBAL_POSITION_INT`가 서로 다른 시점에 도착·유실되어, 방어가 **시각 정렬 없이** 최신
GPS를 오래된(stale) INS와 비교한다. 기체가 원형 순찰을 12 m/s로 도는 동안 ~1.3초 간격이면
약 15m를 이동하므로, 실제 스푸핑이 없어도 겉보기 divergence가 발생한다.

- 시사점: 재밍(가용성 공격)이 위치 무결성 탐지기의 **오탐을 유발**할 수 있다 —
  공격 표면 간 상호작용.
- 개선안: 교차검증 시 두 위치원의 타임스탬프 차이를 확인해 stale 샘플이면 divergence
  판정을 보류하거나, 링크 저하가 동시 관측되면 GNSS 오탐 신뢰도를 하향.
